## Project Overview

Philips TV Remote — web-based remote control for Philips Smart TV (JointSpace API v1, port 1925). Targets Philips 42PFL6158K/12 and similar 6xxx series. Runs as a Python proxy server with a single-page web UI, also packaged as a native iOS app via Capacitor.

## Architecture

```
server.py           ← Python proxy: serves www/, proxies /api/* to TV, handles /discover and /config
www/index.html      ← Single-file web UI (HTML + CSS + JS inline), Capacitor-compatible
ios/App/            ← Xcode project generated by Capacitor (webDir: www)
```

**Request flow:** Browser → `server.py:8888` → `/api/*` stripped to `/*` → Philips TV `:1925`

**Key server endpoints:**
- `GET /discover` — scans local /24 subnet for TVs on JointSpace port (threaded, ~254 concurrent checks)
- `GET/POST /config` — runtime TV IP/port configuration (mutable `tv_config` dict)
- `/api/*` — transparent proxy to `http://{tv_ip}:1925/*`

**Frontend architecture:** The web UI has a setup screen (shown when no TV configured) and a main remote page. TV config is stored server-side via `/config`; the frontend just calls relative URLs. For iOS/file:// contexts, `getServerUrl()` reads `localStorage` for server IP.

## Commands

```bash
# Run server (no dependencies beyond Python 3 stdlib)
python3 server.py

# Run with environment config
TV_IP=192.168.1.100 SERVER_PORT=9000 python3 server.py

# iOS development
npm install
npx cap sync ios
npx cap open ios
```

## Key Design Decisions

- **Zero Python dependencies** — server uses only stdlib (`http.server`, `urllib`, `socket`, `threading`). No pip install needed.
- **ThreadingHTTPServer** — each request runs in its own thread; critical because TV API calls have 5s timeout.
- **Single HTML file** — all CSS/JS is inline in `www/index.html`. No build step, no bundler.
- **Server-side TV config** — TV IP is stored in the Python process (not in frontend), so all clients share the same TV target.
- **Capacitor 8.x** — bridges web UI to native iOS with haptic feedback plugin.

## Deployment

Production server runs on `192.168.31.73` (Wyse 5070, Ubuntu) as systemd service `philips-remote`. Service file uses `Environment=TV_IP=...` for TV address. UFW firewall must allow port 8888.
